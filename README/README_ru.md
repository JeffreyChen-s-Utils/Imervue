## Обзор

Imervue — это рабочая станция для работы с изображениями с GPU-ускорением, поставляемая с **четырьмя вкладками верхнего уровня**:

| Вкладка | Назначение |
|---|---|
| **Imervue** | Просмотр, организация, поиск и пакетная обработка библиотеки изображений |
| **Modify** | Конвейер неразрушающей обработки — слайдеры, кривые, LUT, маски, ретушь, мультиснимки |
| **Paint** | Полнофункциональная студия растрового рисования с кистями, слоями, анимацией, инструментами манги, поддержкой PSD |
| **Puppet** | Самостоятельно разработанный аниматор 2D-марионеток с риггингом — сетки, деформеры, параметры, движения, физика |

Принципы проектирования:

- **Производительность прежде всего** — GPU-ускоренный рендеринг с современными GLSL-шейдерами и VBO
- **Поддержка больших коллекций** — Виртуализированная сетка плиток загружает только видимые миниатюры
- **Плавный опыт** — Асинхронная многопоточная загрузка изображений с предвыборкой
- **Неразрушающая обработка** — Каждая корректировка хранится в рецепте конкретного изображения; файл на диске никогда не перезаписывается, пока вы явно не экспортируете
- **Расширяемость** — Полноценная система плагинов с хуками жизненного цикла / меню / изображений / ввода; MCP-сервер предоставляет AI-ассистентам Qt-независимые инструменты с чистой логикой

## Установка

### Требования

- Python >= 3.10
- GPU с поддержкой OpenGL (доступен запасной программный рендеринг)

### Установка из исходного кода

```bash
git clone https://github.com/JeffreyChen-s-Utils/Imervue.git
cd Imervue
pip install -r requirements.txt
```

### Установка как пакета

```bash
pip install .
```

### Зависимости

| Пакет | Назначение |
|---------|---------|
| PySide6 | Фреймворк GUI на Qt6 |
| qt-material | Тема Material Design |
| Pillow | Обработка изображений |
| PyOpenGL | Привязки к OpenGL |
| PyOpenGL_accelerate | Оптимизация производительности OpenGL |
| numpy | Операции с массивами и кэш миниатюр |
| rawpy | Декодирование RAW-изображений |
| imageio | Ввод-вывод изображений |
| imageio-ffmpeg | Экспорт слайд-шоу в MP4 (H.264 через ffmpeg) |
| defusedxml | Безопасный разбор XML (sidecar-файлы XMP) |

Опциональные (зависят от функций; при отсутствии соответствующая функция корректно отключается):

| Пакет | Назначение |
|---------|---------|
| open_clip_torch + torch | Семантический поиск CLIP (поиск изображений на естественном языке) |
| onnxruntime | AI-апскейл Real-ESRGAN / автотегирование через CLIP ONNX |
| opencv-python | HDR-слияние, сшивка панорам, фокус-стекинг, распознавание лиц, кисть восстановления |
| sounddevice | Синхронизация губ Puppet с микрофона |
| mediapipe | Отслеживание лица Puppet через веб-камеру |

## Использование

### Базовый запуск

```bash
python -m Imervue
```

### Открытие конкретного изображения или папки

```bash
python -m Imervue /path/to/image.jpg
python -m Imervue /path/to/folder
```

### Параметры командной строки

| Параметр | Описание |
|--------|-------------|
| `--debug` | Включить режим отладки |
| `--software_opengl` | Использовать программный рендеринг OpenGL (устанавливает `QT_OPENGL=software` и `QT_ANGLE_PLATFORM=warp`) |
| `file` | (позиционный) Файл изображения или папка для открытия при запуске |

## Imervue — Просмотрщик изображений и библиотека

Вкладка **Imervue** — это стартовая поверхность по умолчанию. Она объединяет просмотрщик изображений с деревом папок, боковой панелью EXIF и инструментами организации библиотеки.

### Просмотрщик

- **GPU-ускоренный рендеринг** через OpenGL (шейдеры GLSL 1.20 с VBO)
- **Пирамида глубокого зума** — многоуровневые плитки 512×512 с передискретизацией LANCZOS, LRU-кэш до 256 плиток / бюджет VRAM 1.5 ГБ, анизотропная фильтрация до 8×
- **Асинхронная загрузка** — многопоточное декодирование с предвыборкой ±3 изображения
- **Виртуализированная сетка миниатюр** — рендерятся только видимые плитки; размер миниатюр настраивается (128 / 256 / 512 / 1024 / авто)
- **Дисковый кэш** — сжатые PNG-миниатюры с инвалидацией на основе MD5 в `%LOCALAPPDATA%/Imervue/cache/thumbnails` (или `~/.cache/imervue/thumbnails`)
- **Воспроизведение анимаций** — GIF / APNG с управлением воспроизведением / паузой / покадровым перемещением / скоростью

### Режимы просмотра

- **Сетка** (по умолчанию) — виртуализированная сетка плиток со всплывающим предпросмотром при наведении (задержка 500 мс)
- **Список (детализированный)** — переключение через `Ctrl+L`; столбцы: Предпросмотр · Метка · Имя · Разрешение · Размер · Тип · Изменено
- **Глубокий зум** — двойной клик по плитке; плавное GPU-панорамирование/зум с мини-картой
- **Разделённый вид** (`Shift+S`) — два изображения рядом
- **Чтение разворотами** (`Shift+D`, `Ctrl+Shift+D` для манги справа налево) — читалка с разворотами
- **Зеркалирование на несколько мониторов** (`Ctrl+Shift+M`) — окно для дополнительного дисплея
- **Театральный режим** (`Shift+Tab`) — скрытие всего интерфейса
- **Диалог сравнения** — Бок о бок / Наложение (слайдер alpha) / Разница (слайдер усиления) / Разделение A|B с перетаскиваемым разделителем
- **Виды Timeline / Calendar / Map** — группировка библиотеки по дате съёмки, просмотр в календаре, отображение геотегированных кадров на Leaflet + OpenStreetMap

### Экранные оверлеи

- RGB-гистограмма (`H`)
- F8 OSD (имя файла / размер / тип), Ctrl+F8 отладочный HUD (VRAM / кэш / потоки)
- Пиксельный вид (`Shift+P`) — при зуме ≥ 400 % отображает сетку пикселей + RGB / HEX каждого пикселя
- Цветовые режимы (`Shift+M`) — Normal / Grayscale / Invert / Sepia через GLSL

### Навигация

- Стрелки, история в стиле браузера (`Alt+←/→`), случайный переход (`X`)
- Навигация между папками (`Ctrl+Shift+←/→`)
- Переход к изображению по индексу (`Ctrl+G`)
- Нечёткий поиск (`Ctrl+F` / `/`)
- **Командная палитра** (`Ctrl+Shift+P`) — нечёткий поиск по всем действиям меню
- Автоматическое зацикливание в конце папки
- Зум щипком на тачпаде + горизонтальный свайп для навигации

### Организация

- **Закладки** — до 5000 путей
- **Оценки** — 0-5 звёзд (`1`–`5`) + сердечко избранного (`0`)
- **Цветовые метки** — флаги красный/жёлтый/зелёный/синий/фиолетовый (`F1`–`F5`)
- **Отбор (Culling)** — трёхпозиционный флаг other XMP-aware photo managers (`P` = принять, `Shift+X` = отклонить, `U` = снять флаг); фильтрация по состоянию; массовое удаление отклонённых
- **Иерархические теги** — древовидные пути вида `animal/cat/british`; потомки сопоставляются автоматически
- **Tags & Albums** с фильтрацией по нескольким тегам AND/OR
- **Умные альбомы** — сохранение запросов на основе правил (расширение / разрешение / оценка / цвет / отбор / теги) и применение одним кликом
- **Стек пар RAW+JPEG** — объединение снимков с одинаковым именем в одну плитку; RAW остаётся доступным как родственный элемент
- **Заметки к изображениям** в боковой панели EXIF — отложенное сохранение, сохраняется между сеансами
- **Лоток подготовки** — корзина между папками, переживающая перезапуски; массовое перемещение / копирование / экспорт
- **Двухпанельный файловый менеджер** — двухпанельный вид с двумя деревьями
- **Сеансы / Макеты рабочей области** — снимок вкладок / выбора / фильтра / геометрии доков в `.imervue-session.json`; сохранение именованных макетов для расположений Browse / Develop / Export
- **Макросы** — запись / воспроизведение пакетов действий оценки / избранного / цвета / тегов (`Alt+M` воспроизводит последний макрос)
- **Бейджи миниатюр + плотность** — цветовая полоса, избранное, закладка, звёзды оценки; отступы Compact / Standard / Relaxed
- **Перетаскивание во внешние приложения** — перетаскивание плитки прямо в Explorer / Chrome / Discord
- **Недавние папки / изображения** отслеживаются; последняя папка автоматически восстанавливается при запуске

### Сортировка и фильтр

- Сортировка по имени / изменению / созданию / размеру / разрешению (по возрастанию или убыванию)
- Фильтр по расширению, цветовой метке, оценке, тегу/альбому, состоянию отбора
- **Расширенный фильтр** — разрешение / размер файла / ориентация / диапазон даты изменения
- **Диалог фильтра по нескольким тегам** с булевой логикой AND / OR

### Поиск

- **Нечёткий поиск по имени файла** с подсветкой подстрок
- **Поиск похожих изображений** — pHash (64-битный DCT) с настраиваемым расстоянием Хэмминга
- **Поиск по библиотеке** — SQLite-индекс с несколькими корнями, поиск по расширению / размеру / размерам / имени файла
- **Семантический поиск (CLIP)** — запросы на естественном языке («золотистый ретривер в снегу») через кэшированные embedding; корректно недоступен при отсутствии `open_clip_torch` + `torch`
- **Автотегирование** — эвристическая классификация с опциональным апгрейдом до CLIP ONNX

### Метаданные

- **Боковая панель EXIF** со сворачиваемыми группами + встроенная строка из 0-5 звёзд
- Диалог **редактора EXIF**
- Диалог **информации об изображении** (размеры / размер файла / даты)
- **XMP sidecar-файлы** (`.xmp`-компаньоны) — двусторонняя синхронизация оценки / заголовка / описания / ключевых слов / цветовой метки для совместимости с other XMP-aware photo managers (безопасный XML через `defusedxml`)
- **Редактор GPS-геотегов** — чтение существующих EXIF GPS, запись новых широты/долготы через piexif (JPEG)
- **Пакетное переименование по токенам** — шаблоны с предпросмотром в реальном времени, например `{date:yyyymmdd}_{camera}_{counter:04}{ext}`
- **Экспорт метаданных CSV / JSON** — одна строка на изображение, включая отбор / оценку / теги / заметки

### Дополнительные инструменты (вкладка Imervue — пакетная обработка)

Доступ из меню **Tools**; организованы в сгруппированные по функциям подменю:

- **Пакетные** — Преобразование формата · Очистка EXIF · Санитайзер изображений (повторный рендеринг для удаления скрытых данных) · Органайзер изображений (сортировка по подпапкам по дате / разрешению / типу / размеру) · Пакетное переименование по токенам
- **AI / Эвристика** — AI-апскейл изображений (Real-ESRGAN x2 / x4 + ONNX Runtime CUDA/DML/CPU) · Поиск дубликатов · Поиск похожих · Автотегирование · Распознавание лиц (каскады Хаара)
- **Библиотека и метаданные** — Поиск по библиотеке · Умные альбомы · Иерархические теги · Экспорт метаданных · XMP sidecar · GPS-геотеги

### Интеграция с системой

- Контекстное меню Windows по правому клику **Open with Imervue** (ассоциация файлов через реестр)
- Мониторинг папок через `QFileSystemWatcher` (автообновление при изменениях)
- Система уведомлений Toast (info / success / warning / error)
- Система плагинов с онлайн-загрузчиком (см. [Система плагинов](#система-плагинов))

## Modify — Неразрушающая обработка

Вкладка **Modify** — рабочая станция обработки. Каждая корректировка живёт в **рецепте** конкретного изображения, хранимом рядом с файлом — исходные пиксели на диске никогда не перезаписываются, пока вы явно не нажмёте **Export** или **Save As**.

### Слайдеры обработки

- Баланс белого — температура / оттенок
- Тональные диапазоны — тени / средние тона / света
- Экспозиция / контраст / насыщенность / сочность
- Кадрирование, поворот, горизонтальное / вертикальное отражение
- Все правки остаются неразрушающими и проходят туда-обратно через хранилище рецептов

### Кривые и LUT

- **Редактор тоновой кривой** — перетаскиваемая RGB-кривая плюс отдельные каналы R / G / B с монотонной кубической интерполяцией
- **Применение .cube LUT** — загрузка любого Adobe 3D LUT (до 64³), трилинейная интерполяция, смешивание со слайдером интенсивности
- **Разделённое тонирование** — оттенок + насыщенность теней / светов на основе флагов с осью балансировки

### Локальные корректировки

- **Маски кистью / радиальным / линейным градиентом** с дельтами экспозиции / яркости / контраста / насыщенности / баланса белого для каждой маски + слайдер растушёвки
- Маски смешиваются неразрушающим образом через конвейер обработки

### Ретушь и трансформация

- **Лечащая кисть** — круглые точки, инпейнтинг OpenCV (Telea или Navier-Stokes)
- **Штамп клонирования** — Shift+клик по источнику, растушёванный перенос в место назначения
- **Кадрирование / Выпрямление** — нормализованный прямоугольник кадрирования плюс выпрямление под произвольным углом с автоматическим обрезом до наибольшего внутреннего прямоугольника
- **Автовыпрямление** — определение горизонта / вертикали через Hough-линии
- **Коррекция объектива** — чистая numpy-радиальная дисторсия (бочка / подушка), компенсация виньетирования, поканальная хроматическая аберрация
- **Шумоподавление / Резкость** — двусторонний шумодав с сохранением краёв + повышение резкости через unsharp-mask
- **Небо / Фон** — замена обнаруженного неба на градиент или удаление фона (прозрачный или белый заполнитель); опциональный апгрейд через `rembg` / U²-Net

### Мультиснимки

- **HDR-слияние** — объединение брекетированных экспозиций через OpenCV Mertens fusion (с предварительным выравниванием AlignMTB)
- **Сшивка панорам** — OpenCV `Stitcher` в режиме panorama или scans, автообрез чёрной рамки
- **Фокус-стекинг** — карта фокуса по дисперсии лапласиана + гауссово смешивание с опциональным выравниванием ECC

### Вывод

- **Наложение водяного знака** — текст или изображение, 9 точек привязки, прозрачность, масштаб; применяется только при экспорте
- **Пресеты экспорта** — конвейеры одного клика Web 1600 / Print 300 dpi / Instagram 1080
- **Save As / Export** — PNG / JPEG / WebP / BMP / TIFF со слайдером качества для форматов с потерями
- **Пакетные операции** — переименование, перемещение/копирование, поворот выделенных изображений
- **Contact Sheet PDF** — многостраничная сетка с подписями (A4 / A3 / Letter / Legal)
- **Web Gallery HTML** — самодостаточная папка с `index.html` + JPEG-миниатюрами + встроенным lightbox
- **Слайд-шоу MP4** — видео H.264 с настраиваемым FPS / временем удержания на изображение / переходами с плавным затуханием (`imageio-ffmpeg`)
- **Макет для печати** — многостраничный PDF-лист с настраиваемым размером страницы / ориентацией / сеткой / полями / корешком / метками обрезки
- **Программная цветопроба** — загрузка ICC-профиля, симуляция целевого цветового охвата, выделение пикселей вне охвата пурпурным

## Puppet — 2D-анимация с риггингом

Вкладка **Puppet** — это с нуля разработанная система 2D-анимации марионеток с риггингом. Она делает то же, что и Live2D (риги с сеточной деформацией, параметры, движения, физика, выражения, поза, синхронизация губ, отслеживание лица с веб-камеры), но **без проприетарного SDK**, **без `live2d-py`** и с полностью открытым форматом файлов `.puppet`, задокументированным в `Imervue/puppet/FORMAT.md`.

> **Полное руководство**: [`puppet_guide.md`](../puppet_guide.md) охватывает
> сквозной процесс как для прямой трансляции (OBS / NDI / виртуальная камера),
> так и для производства анимации (запись / редактирование таймлайна / экспорт
> MP4). Китайские версии в
> [`puppet_guide.zh-TW.md`](../puppet_guide.zh-TW.md) и
> [`puppet_guide.zh-CN.md`](../puppet_guide.zh-CN.md).

### Формат файла

`.puppet` — это zip-контейнер:

- `puppet.json` — манифест (drawables, deformers, parameters, motions, pose groups, parts, hit areas)
- `textures/*.png` — текстуры атласа
- `motions/*.json` — keyframe-треки
- `expressions/*.json` — оверлеи параметров
- `physics.json` — конфигурация Verlet-физики

На основе JSON, поддаётся человеческому diff, без проприетарного бинарного формата.

### Рендерер

`QOpenGLWidget` с рисованием текстурированных треугольников через vertex-array в порядке draw_order, режимами смешивания на каждый drawable (normal / additive / multiply), эксклюзивностью pose-групп, ортогональной проекцией в пространстве изображения, фоном-шахматкой прозрачности с тайлингом GL_REPEAT, зум колесом + панорамирование средней кнопкой мыши. Оптимизирован для крупных ригов — March 7th (307 drawables / 2965 vertex morphs) работает на 60 FPS на CPU.

### Авторинг

- **Импорт PNG** → автоматическая генерация сетки триангулированных ячеек с учётом alpha
- Действия на панели инструментов **Add Rotation Deformer** (anchor + угол) / **Add Warp Deformer** (bezier-решётка rows × cols)
- **Add Parameter** → задание ключевых форм на крайних значениях слайдера через **Set Key** в доке параметров
- **Редактор сетки** — переключение Edit Mesh для перетаскивания вершин; клики в пределах 8 px привязываются к ближайшей
- **Save As…** записывает весь риг в `.puppet` zip

### Выполнение

- **Риг параметров** — каждый параметр хранит список ключей, отображающий значение слайдера в частичный снимок формы деформера; рантайм семплирует и выполняет линейную интерполяцию по полям
- **Воспроизведение движений** — нижний док со списком движений + Play / Pause / Stop / Loop / scrub; семплер кривых учитывает сегменты `linear`, `stepped`, `inverse-stepped`, `cubic-bezier` (решение time → param методом Ньютона); fade-in / fade-out для каждого движения
- **Выражения** — стек оверлеев параметров `additive` / `multiply` / `overwrite`
- **Группы поз** — взаимоисключающая видимость drawable (смена оружия, варианты формы рта)
- **Физика** — Verlet-цепи маятников для волос / одежды / лент; входной параметр перемещает якорь цепи, гравитация + затухание + пружины на каждую частицу возвращают к состоянию покоя
- **Vertex morphs** — линейное смешивание в стиле Cubism между rest и ±extreme дельтами; векторизованный numpy на каждый кадр при 60 FPS
- **Ключи непрозрачности** — кривые alpha, управляемые параметрами; позволяет сеткам альтернативных поз появляться / исчезать при срабатывании параметра жеста

### Живой ввод

- Перетаскивание курсора → параметры угла головы
- Автоматическое моргание по косинусоидальной кривой open → close → open
- Синхронизация губ с микрофона через `sounddevice` RMS → `ParamMouthOpenY` (опциональная зависимость)
- Отслеживание лица с веб-камеры через OpenCV + MediaPipe FaceMesh → yaw / pitch / roll головы + открытие глаз / рта (опциональные зависимости)
- Запись пользовательских движений — захватывает значения параметров с частотой 30 Гц, пока вы двигаете слайдеры / смотрите в веб-камеру / запускаете физику; запекает в Motion с линейными сегментами, готовый к воспроизведению / зацикливанию / сохранению

### Совместимость с Cubism

**Cubism Native SDK** может быть подключён (пользовательский DLL — Free Material License от Live2D запрещает перераспространение) для конвертации любой модели `.moc3` в `.puppet` zip. Конвертер выполняет проход sample-and-reconstruct, который захватывает как vertex-morph дельты, так и переходы видимости, управляемые параметрами, поэтому переключатели жестов (знак мира / закрытие лица / фото …) переживают конвертацию в целости.

### Вывод

- **Capture frame…** сохраняет PNG текущего холста через `glReadPixels`
- **Record…** переключает цикл кадров 30 FPS в GIF / WebM / MP4 через `imageio`
- **Виртуальная камера** — представляет холст puppet как системную веб-камеру
- **NDI-вывод** — транслирует puppet как NDI-источник в локальной сети
- **VTube Studio API server** — опциональный WebSocket API для VTS-совместимых клиентов

### Прямая трансляция в OBS

Два поддерживаемых пути. Выбирайте A для «просто работает», B — если хотите минимальную задержку и лучшее качество в быстрой локальной сети.

#### A. Виртуальная камера (проще всего)

Холст puppet появляется как веб-камера, которую OBS подхватывает через свой стандартный источник Video Capture Device.

1. `pip install pyvirtualcam`
2. Установите драйвер для платформы:
   - **Windows**: OBS Studio 26+ поставляет драйвер *OBS Virtual Camera*.
     После установки OBS откройте его один раз и нажмите **Start
     Virtual Camera** на нижней правой панели — это регистрирует
     драйвер на уровне системы, чтобы `pyvirtualcam` смог его найти.
   - **macOS**: OBS для Mac поставляет системное расширение OBS Virtual Camera.
     Первый запуск предложит включить его в разделе
     System Settings → Privacy & Security.
   - **Linux**: `sudo modprobe v4l2loopback exclusive_caps=1 card_label="Imervue"` (предварительно установите `v4l2loopback-dkms`).
3. На вкладке Puppet откройте ваш риг, затем включите **Output > Virtual
   camera**. В строке состояния отображается точное имя устройства для выбора.
4. В OBS: **Sources > + > Video Capture Device**, выберите устройство,
   указанное на шаге 3 (обычно *OBS Virtual Camera*).

Imervue ограничивает наибольшую сторону потокового вывода 1080 px,
чтобы Cubism-нативные холсты (у March 7th — 3503×7777) не отвергались
драйвером виртуальной камеры DirectShow. Соотношение сторон
сохраняется; OBS может дополнительно масштабировать при необходимости.

##### Почему фон пурпурный? (и как его убрать)

Виртуальные камеры работают через **DirectShow** (Windows) / **AVFoundation**
(macOS) / **v4l2loopback** (Linux). Все три транспорта
**только RGB — без альфа-канала**. Источник *Video Capture Device* в OBS
обрабатывает то, что отправляет камера, как непрозрачный RGB, поэтому
какой бы цвет Imervue ни поставил за персонажа, именно его OBS и отобразит.

Imervue выбирает **пурпурный `#FF00FF`** в качестве этого фона, потому что
это индустриальный стандартный цвет для chroma-key: он практически
никогда не встречается в тонах кожи, волосах или цветах глаз, поэтому
порог chroma-key можно открыть широко без задевания персонажа.

Чтобы убрать пурпурный в OBS:

1. Правый клик по источнику *Video Capture Device*, который вы добавили → **Filters**
2. В нижней левой панели **Effect Filters** → **+** → **Color Key**
3. Настройте:
   - **Key Color Type**: `Custom Color`
   - **Custom Color**: HEX `FF00FF` (или R = 255 / G = 0 / B = 255)
   - **Similarity**: начните с `80`, повышайте до `200–300`, если
     остаются пурпурные края. Выше = агрессивнее удаление.
   - **Smoothness**: `30–50` смягчает край, чтобы вырез не выглядел
     жёстким / пикселизированным.
4. Закройте диалог. OBS прикрепляет фильтр к источнику, так что
   в следующий раз, когда вы включите виртуальную камеру,
   chroma-key применится автоматически.

Если у персонажа в палитре есть пурпурный (необычно, но возможно
для костюма / арта реквизита), chroma-key съест и эти пиксели тоже.
Переключитесь на путь NDI ниже — NDI передаёт альфа-канал напрямую,
поэтому никакой chroma-key не нужен.

**Устранение неполадок: я всё ещё вижу пурпурный в OBS**

- Убедитесь, что фильтр Color Key прикреплён к источнику **Video Capture
  Device**, а не к сцене. Фильтры на источнике перемещаются
  вместе с ним; фильтры на сцене применяются сверху после того, как
  источник уже отрендерился.
- Проверьте, что hex — это в точности `FF00FF` — `FF00FE` или подобный
  не охватит все пурпурные пиксели.
- Повысьте *Similarity* до `300`, если на контуре персонажа есть
  тонкий ореол пурпурных пикселей. Края возникают из-за
  интерполяции GL_LINEAR относительно пурпурного фона; более
  широкая толерантность similarity их съедает.

#### B. NDI (минимальная задержка, профессиональный уровень)

NDI (Network Device Interface от Newtek) переносит puppet по
локальной сети с задержкой менее 50 мс и сохранением альфа-канала.

1. Скачайте и установите **NDI Tools** с
   <https://ndi.video/tools/> (включает NDI runtime).
2. `pip install ndi-python`
3. Установите плагин **obs-ndi** в OBS:
   <https://github.com/obs-ndi/obs-ndi/releases>
4. На вкладке Puppet включите **Output > NDI output**. В строке
   состояния отображается имя NDI-источника (по умолчанию *Imervue Puppet*).
5. В OBS: **Sources > + > NDI Source**, выберите источник из
   шага 4.

NDI вещает в том же разрешении с ограничением 1080, что и путь A,
но доставляет RGBA — off-screen рендеринг создаёт прозрачный
фон за пределами персонажа, NDI передаёт альфа-канал
без изменений, и OBS / vMix композитят puppet прямо поверх вашей
сцены без какого-либо прохода chroma-key.

#### C. Захват окна (запасной)

OBS **Sources > + > Window Capture** может захватить окно Imervue
напрямую, без дополнительных зависимостей. Качество ниже, и вам
придётся самостоятельно обрезать интерфейс, но это работает на
заблокированных машинах, где нельзя устанавливать драйверы.

### Демо

Готовый риг находится в [`examples/puppet/march_7th.puppet`](../examples/puppet/march_7th.puppet) — персонаж Cubism Live2D из 307 drawable, конвертированный в репозитории. Откройте через **Open Puppet…**, чтобы увидеть, как риг загружается по центру; нажмите любой из 18 движений (группа Idle + группа Gesture), чтобы воспроизвести. Жесты охватывают знак мира, закрытие лица, фото, румянец, тёмное лицо, плач, пот, звёзды, падающую звезду — каждый именованный жест, определённый в риге.

1. Создайте папку внутри `plugins/` в корне проекта

## MCP-сервер

Imervue поставляет встроенный сервер [Model Context Protocol](https://modelcontextprotocol.io), чтобы AI-ассистенты (Claude Code / Desktop, Cursor, Cline, …) могли вызывать чистые логические помощники проекта без запущенного GUI. Без Qt; одна команда:

```sh
python -m Imervue.mcp_server
```

### Инструменты

| Инструмент | Назначение |
|------|---------|
| `list_images` | Список файлов изображений в папке (опциональная рекурсия) |
| `read_image_metadata` | Размеры, формат, EXIF и XMP sidecar |
| `read_xmp_tags` | Быстрый путь только для XMP: оценка, метка, ключевые слова |
| `convert_format` | Конвертация между PNG / JPEG / WebP / TIFF / BMP |
| `puppet_from_png` | Построение `.puppet` рига из PNG (автосетка + стандартные параметры) |
| `puppet_inspect` | Открыть `.puppet` и вернуть его инвентарь |

### Подключение

Репозиторий поставляет `.mcp.json` в корне для автообнаружения Claude Code. Для Desktop / других клиентов добавьте это в `claude_desktop_config.json` (или эквивалент):

```json
{
  "mcpServers": {
    "imervue": {
      "type": "stdio",
      "command": "python",
      "args": ["-m", "Imervue.mcp_server"]
    }
  }
}
```

Полный интерфейс протокола в разделе MCP документа [docs/en/index.rst](../docs/en/index.rst).

### Конвейер рендеринга (вкладка Imervue)

1. `GPUImageView` расширяет `QOpenGLWidget`
2. Две GLSL 1.20 программы (textured quads + прямоугольники сплошного цвета)
3. LRU-кэш текстур — лимит 256 плиток, бюджет VRAM 1.5 ГБ
4. Многоуровневая пирамида плиток, построенная с LANCZOS при размере плитки 512 × 512
5. Анизотропная фильтрация до 8×, когда оборудование её поддерживает
6. Запасной программный рендеринг при сбое компиляции шейдеров

### Кэш миниатюр

- **Ключ**: MD5 от `{path}|{mtime_ns}|{file_size}|{thumbnail_size}`
- **Формат**: Сжатый PNG (`compress_level=1` — быстрая запись, малый объём)
- **Расположение**: `%LOCALAPPDATA%/Imervue/cache/thumbnails` (Win) или `~/.cache/imervue/thumbnails` (Linux/macOS)
- **Инвалидация**: Автоматическая при изменении метаданных файла

### Рендеринг Puppet (вкладка Puppet)

- `QOpenGLWidget` с `glDrawElements` + клиентскими vertex array
- Для каждого drawable: rest-вершины кэшируются как float32 numpy; vertex morph векторизированы; топологическая сортировка деформеров вынесена из цикла по drawable
- Фон прозрачности — это текстура с тайлингом GL_REPEAT 2×2 (до оптимизации было 100k+ immediate-mode quads)
- Конвертер Cubism создаёт кривые opacity_keys наряду с vertex-morph дельтами, поэтому переходы видимости, управляемые параметрами, переживают конвертацию `.moc3 → .puppet`
